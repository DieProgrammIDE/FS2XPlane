FS2Xplane

FS2004 Add-on Scenery converter for X-Plane

Overview

This application converts MS Flight Simulator 2004 add-on scenery packages to X-Plane® scenery packages. The application only runs on Windows, however the resulting X-Plane scenery packages will work on any platform supported by X-Plane 8.30 and later, provided that standard ENV or DSF scenery for the area is also installed. Some users have also reported good results under Virtual PC on Mac and under Wine on Linux.

See the license section below before using this application.

The following scenery features are converted:

  • FS2004-style runways and taxiways.
  • Procedural scenery.
  • Exclusion of generic objects.

See the limitations section for details of features that are not converted.

This application works best with scenery packages that have been specifically designed for FS2004 - ie containing airports authored in AFCAD2 and objects authored in Gmax. The application also works with FS2004 scenery packages that contain old-style BGL procedural scenery, but produces less efficient X-Plane scenery (mostly this is due to the fact that older MSFS scenery packages tend to use lots of small texture files).

Setting Up

  • Extract the contents of this zip file to a folder. The rest of these notes refer to this as the FS2XPlane folder.
  • Download the "DSF2Text" package from here. Copy the program DSFTool.exe from this package to the FS2XPlane folder.

Usage

Short version:

Usage is:

fs2x <location of MSFS scenery package> <location for new X-Plane scenery package>

The location for the new X-Plane scenery package must be a non-existent or empty folder directly under the X-Plane Custom Scenery folder.

Long version:

  • Find the location of the scenery package that you want to convert. For scenery packages that have been installed in FS2004 this will most likely be a folder under C:\Program Files\Microsoft Games\Flight Simulator 9\Addon Scenery\
  • Find the location of the Custom Scenery folder in X-Plane. By default this will be C:\X-Plane\Custom Scenery
  • Open a "Command Prompt" window:
    • Click Start » Run...
    • Type cmd
    • Press Enter.
  • Change directory to the FS2XPlane folder:
    • In the command prompt window type cd Space
    • Drag and drop the FS2XPlane folder on to the command prompt window
    • Press Enter.
  • Run the application:
    • In the command prompt window type fs2x Space
    • Drag and drop the MSFS scenery package folder on to the command prompt window
    • Press Space
    • Drag and drop the X-Plane Custom Scenery folder on to the command prompt window
    • Type \ followed the name that you want to give to the X-Plane scenery package
    • Press Enter.

The application runs for a while and displays the names of the files that it is processing. It will
stop and display an error and if it encounters a fatal problem during the conversion. Otherwise, on completion, the application will say "Done" and an error log may be displayed. The error log is described below.

Files produced

The application produces the following files in the X-Plane scenery package folder:

File name Description
errors.txt Produced if there were any problems with the conversion. Discussed below.
You need not include this file in your scenery package.
Earth nav data\apt.dat Airport, runway, taxiway and COM data.
Earth nav data\nav.dat ILS, DME and VOR navaids.
Note that X-Plane will not pick up entries from this file. You need to use this procedure if you want to update X-Plane's master navaid database. You'll need to justify any changes based on real-world data.
Earth nav data\*\*.dsf "Overlays" for 1ºx1º areas of the planet, containing placements of scenery objects.
Earth nav data\*\*.txt Source of the corresponding .dsf files.
You need not include these files in your scenery package.
objects\*.obj Converted scenery objects. These are named as follows:
  • Apron edge lights: ApronLight.obj
  • Library objects: <id>[-<texture name>][_<scale>].obj
  • Taxiway centerline lights: TaxiwayLight.obj
  • Taxiway signs: TaxiwaySign-<label>[_<scale>].obj
  • Procedural scenery: <BGL name>[-<texture name>][-number].obj
textures\*.png Textures used by the converted scenery objects, converted to PNG format.

Recommended X-Plane rendering options

Since requests in the MSFS scenery to flatten areas are currently ignored, you may need to unset the "allow sloped runways" rendering option.

Recommended rendering options are:

  • world detail distance: default or low
  • allow sloped runways: unchecked
  • compress textures to save VRAM: checked

Limitations

The following MSFS scenery data are simplified and will need rework in X-Plane:
  • FS2004-style aprons are drastically simplified.
  • "Stock" objects are converted as empty placeholders.
  • Animated (including "billboard") objects are converted as static objects.
  • Many MSFS scenery packages supply ground textures as scenery objects on top of flattened terrain, instead of as proper terrain data. This doesn't work so well in X-Plane.

The following MSFS scenery data are discarded:

  • Data that X-Plane does not understand, eg airspace boundaries.
  • FS2004-style taxiway/apron lines and other markings (this may be addressed in a future X-Plane release).
  • Flattening of terrain regions.
  • Terrain and landclass data (this may be addressed in a future release).
  • Generic buildings.
  • "hard" surfaces.
  • Hold-short points (this may be addressed in a future release).
  • Seasonal textures (this may be addressed in a future X-Plane release).
  • Areas that make use of some of the more obscure pre-FS2002 BGL opcodes.

<Path> is not a folder		 The location that you specify for the source MSFS scenery should be an existing folder.
<Path> is not empty
<Path> is not a sub-folder of "Custom Scenery"
 The location that you specify for the new X-Plane scenery package should be a non-existent or empty folder directly under X-Plane's Custom Scenery folder (which must itself exist).
Can't find "DSFTool.exe"
 See above.
Can't write <filename>
 Are you out of disk space?
Do you have write access to the target folder?
Perhaps your filesystem truncates file names?
Found duplicate definition of runway <n> in <filename> Some scenery packages supply alternative airport layouts. This application should be run using only one of the alternatives.

No data found! The application was unable to find and convert any MSFS scenery.

The application will produce a log if it encountered problems during the conversion:

Error Description and remedial action
Absolute altitude (<n>m) for object <id> at [<lat>, <lon>] in <filename> This object was placed in MSFS at <n> metres above mean sea level (MSL). In X-Plane, all scenery objects are placed at ground level.
You will likely need to edit the converted object to make it appear correctly in X-Plane.
Can't create texture <filename> from <source filename> Conversion of the texture file from R8 or BMP format to PNG format failed for some reason.
Objects that use this texture will appear grey in X-Plane. You can fix this by converting the texture by hand.
Can't find an airport for <thing> at [<lat>, <lon>] The application couldn't find an airport in the MSFS scenery package within 10NM of this beacon or windsock. In X-Plane, beacons and windsocks must be attached to an airport, and within 10NM of the main runway.
The beacon or windsock was omitted.
Can't parse <filename> This is most likely caused by a corrupt BGL file.
This file will not be converted.
Can't parse area #<n> in <filename> This is most likely caused by the presence of obscure pre-FS2002 opcodes in the source BGL file.
This area will likely be incompletely converted, but any other areas in the same file will be OK.
Can't parse object <id> in <filename> This is most likely caused by the presence of obscure pre-FS2002 opcodes in the source library.
This library object will likely be incompletely converted, but any other objects in the same library will be OK.
Can't read <filename> This is most likely caused by a corrupt BGL file.
This file will not be converted.
Non-zero altitude (<n>m) for object <id> at [<lat>, <lon>] in <filename> This object was placed in MSFS at <n> metres above ground level (AGL). In X-Plane, all scenery objects are placed at ground level.
You will likely need to edit the converted object to make it appear correctly in X-Plane.
Non-zero pitch/bank (<pitch>/<bank>) for object <id> at [<lat>, <lon>] in <filename> This object was placed in MSFS with non-zero pitch and/or bank. In X-Plane, all scenery objects are placed upright.
You will likely need to edit the converted object to make it appear correctly in X-Plane.
Object <id> is a built-in object (<name>) The scenery package used a built-in "stock" MSFS scenery object named <name> and with GUID <id>. An empty placeholder object has been created and placed in the X-Plane scenery.
You should either remove this object from the X-Plane scenery or provide a replacement for the placeholder object.
Object <id> not found The scenery package used a library object that was not included in the scenery package.
You should read the installation notes that came with the scenery package to identify the missing library that supplies the object. Then copy the missing library into the scenery package and run the application again.
Skipping duplicate of taxiway at [<lat>, <lon>] in <filename> Some scenery packages supply alternative airport layouts.
Read the installation notes that came with the scenery package, delete the alternative layout, and run the application again.
Skipping misc data in file <filename> This is most likely caused by a request in the MSFS scenery package to flatten an area.
The request is ignored.
Skipping pre-FS2004 runways, taxiways and/or roads in <area# or object id> in <filename> Runways, taxiways and/or roads were found in the old pre-FS2004 format. This application only handles FS2004-format runways and taxiways (ie those authored with AFCAD2 or similar).
The runways, taxiways and/or roads are ignored.
Skipping pre-FS2002 scenery in <area# or object id> in <filename> Old pre-FS2002 opcodes were found.
The old-style scenery is not converted, but other scenery in this area or library object will be OK.
Skipping terrain data in file <filename> The scenery package includes terrain and/or landclass data.
This data is ignored.
Skipping traffic data in file <filename> The scenery package includes AI traffic data.
This data is ignored.
Texture <filename> not found The scenery package used a texture file that was not included in the scenery package. Either:
  • The file is one of the "stock" textures that are bundled with the MSFS program. Objects that use this texture will appear grey in X-Plane. You can fix this by providing a replacement for this texture.
  • This scenery package relies on another package, which supplies the missing file. Read the installation notes that came with the scenery package to identify the missing package. Then copy the missing package into this package and run the application again.
Unsupported generic building at [<lat>, <lon>] in <object or filename> X-Plane doesn't support the concept of "generic buildings". The nearest X-Plane equivalent is "library" objects. However there is no simple mapping between the two concepts.
The building is ignored. You will need to add a new or library object at this location.
Unsupported request for object <id1> within object <id2> in <filename> X-Plane doesn't support nested objects within library objects.
The second object is ignored.
