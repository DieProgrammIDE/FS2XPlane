MS FS Add-on Scenery converter for X-Plane®

Overview

This application converts MS Flight Simulator 2004 and FSX add-on scenery packages to X-Plane DSF overlay scenery packages. The resulting X-Plane scenery packages work under X-Plane 8.60 and later on Windows, Mac OS and Linux. Standard "ENV" or "DSF" global scenery for the relevant areas must also have been installed.

See the license section below before using this application.

The following scenery features are converted:

  • FS2004 and FSX style runways and taxiways.
  • Procedural scenery.
  • Exclusion of default/generic objects.

See the limitations section for details of features that are not converted.

This application works best with scenery packages that have been specifically designed for FS2004 or FSX - ie containing airports authored in AFCAD2 and objects authored in Gmax® or 3ds Max®. The application also works with FS2002 and older scenery packages that contain old-style BGL procedural scenery, but produces poorer and less efficient X-Plane scenery (this is mostly due to the fact that older MSFS scenery packages tend to use lots of small texture files).

Requirements

Windows
Requires Windows 2000 or later.
 
Mac OS
Requires Mac OS 10.3.9 or later.
If you have an Intel Mac then you should install X11 from the your Mac OS DVD or CD 3, and also download Darwine (x86) - drag the folder Darwine to your Applications folder to install it. This will allow you to run MSFS scenery package installer programs and will also in some cases allow FS2XPlane to produce more complete X-Plane scenery.
 
Linux
Requires "Wine". Wine is shipped with most distros, or you can download a binary.
The GUI application additionally requires the wxPython runtime. wxPython (aka python-gtk2) is shipped with most distros, or you can download a binary (or just stick with the command-line program).

Setting Up

Run the FS2XPlane application.

Windows
If you don't have FS2004 or FSX installed then FS2XPlane creates a fake FS2004 installation in Program Files\Microsoft Games\Flight Simulator 9 and sets up the subfolders and registry entries that some MSFS scenery package installer programs require.
 
Mac OS and Linux
FS2XPlane creates a fake FS2004 installation in the folder FS2004 inside of your home folder, and sets up the folders and Wine registry entries that some MSFS scenery package installer programs require.
(If you want to move or rename the FS2004 folder then you must update the UNIX symbolic link ~/.wine/dosdevices/c:/Program Files/Microsoft Games/Flight Simulator 9 to point to the new location).

Usage

FS2XPlane can operate in one of two ways - it can convert a MSFS scenery package or it can extract all the objects in MSFS scenery libraries.

Converting a scenery package
  • Run the FS2XPlane application.
  • Download the scenery package that you want to convert. There are two kinds of scenery packages; those that come with an installer program and those that you install manually:
    • Installer program:
    Double-click on the installer program to run it (unless you have a PPC Mac, in which case you can't use this scenery package).
    Let the program put the scenery in the default location.
    The installer program puts the scenery in a folder somewhere inside of your MSFS installation. Find this subfolder (it may be inside the Addon Scenery folder).
    Manual installation:
    Read the installation notes for the scenery package. Most packages require you to unzip the package into a new folder under the Addon Scenery folder inside of your MSFS installation.
  • Read the installation notes for the scenery package. Look for:
    • Whether the package depends on a scenery library. If so, download and follow the installation notes for this additional scenery library. See the notes below if your package depends on the "rwy12" library.
    • Whether the packages supplies multiple alternative airport definitions (these are sometimes described as "AFCAD files" in the installation notes). If so, the installation notes may advise you to copy one of these files to the scenery folder inside the installed package.
  • In FS2XPlane, supply the location of the folder that contains the MSFS scenery package.
  • Supply the location of any additional MSFS libraries that the package depends on. Specify the Addon Scenery folder to have FS2XPlane search through all installed libraries (recommended), or leave this blank for none.
  • Specify the location for the new X-Plane scenery package. This should be a non-existent or empty folder directly under X-Plane's Custom Scenery folder.
  • Select the season for converted textures. (Not all MSFS scenery packages supply seasonal textures, in which case this setting will be ignored).
  • Press Convert.

The application runs for a while and displays the names of the files that it is processing. It will stop and display an error if it encounters a fatal problem during the conversion. Otherwise, on completion, the application will say "Done" and may display a summary of problems that it encountered. This summary is described below.

Extracting library objects

FS2XPlane can extract objects from "BGL", "Rwy12" and "EZ-Scenery" libraries. It cannot read uncompiled "API", "SBuilder", "SCM" or "MDL" files (and neither can MSFS).

  • Download the scenery library or libraries that you want to convert.
    • For the "rwy12" library see the notes below.
    • Otherwise follow the installation notes in the scenery library.
  • Run the FS2XPlane application.
  • Select the "Just extract library objects" option.
  • Ignore the "MSFS scenery location" - it's not used.
  • Supply the location where you put the MSFS scenery libraries.
  • Specify the location for the new X-Plane scenery objects. This should be a non-existent or empty folder.
  • Select the season for converted textures. (Not all MSFS scenery libraries supply seasonal textures, in which case this setting will be ignored).
  • Press Convert.
Command line

Windows/Linux:

fs2xp [options] "location of MSFS scenery package" "location for new X-Plane scenery"

The following options are supported:

  • Additional scenery libraries: -l "location of additional MSFS scenery libraries"
  • Season for textures: -s Spring or Summer or Autumn or Winter
  • Just extract library objects: -x

Recommended X-Plane rendering options

Recommended rendering options are:

  • world detail distance: "default" or "low"
  • number of objects: anything other than "none"
  • allow sloped runways: unchecked
  • compress textures to save VRAM: checked

Files produced

The application produces the following files in the X-Plane scenery package folder:

File name Description
summary.txt Produced if there were any problems with the conversion. Discussed below.
You need not include this file in your scenery package.
readme*, *.pdf Documentation files copied from the MSFS scenery package.
Earth nav data\apt.dat Airport, runway, taxiway and COM data.
Earth nav data\nav.dat ILS, DME and VOR navaids.
Note that X-Plane will not pick up entries from this file. Use this procedure if you want to update X-Plane's master navaid database. You'll need to justify any changes based on real-world data.
Earth nav data\*\*.dsf Overlay DSFs for 1ºx1º areas of the planet, describing the placement of scenery objects.
objects\*.obj Scenery objects. These are named as follows:
  • Library objects: <id>[-<texture name>][_<scale>].obj
  • Procedural scenery: <BGL name>[-<texture name>][-number].obj
objects\*.pol Draped polygon definitions.
objects\*.png Textures used by the scenery objects and polygons, converted to PNG format.

Limitations

The following MSFS scenery data are simplified and will need rework in X-Plane:
  • Most "stock" MSFS objects are ignored.
  • Animated (including "billboarded") objects are converted as static objects.
  • Partially transparent colours are converted as fully transparent or opaque. (But transparent textures work fine).
  • Many MSFS scenery packages supply ground-level textures (eg taxiways) as scenery objects on top of flattened terrain. This can lead to these textures "floating" above the ground in X-Plane.
  • "Generic" MSFS buildings are not textured.

The following MSFS scenery data are discarded:

  • FSX photoscenery.
  • Scenery made by FSX MDL design tools.
  • Data that X-Plane does not understand, eg airspace boundaries.
  • Terrain and landclass data.
  • "Hard" surfaces above ground-level.
  • Areas that make use of some of the more obscure pre-FS2002 BGL opcodes.

The application will produce a log if it encountered problems during the conversion. You can fix some of these problems by downloading libraries or editing the converted objects:

Absolute altitude (<n>m) for object <id> at (<lat>, <lon>) in file <filename> This object was placed in MSFS at <n> metres above mean sea level (MSL). In X-Plane, all scenery objects are placed at ground level.
You may need to edit the converted object to make it appear at the correct distance above the ground in X-Plane.
Can't convert texture <filename> Conversion of this texture file from R8, BMP or DDS format to PNG format failed for some reason.
Objects that use this texture will appear grey in X-Plane. You can fix this by converting the texture by hand.
Can't find an airport for <beacon, taxiway or windsock> at (<lat>, <lon>) The application couldn't find an airport in the MSFS scenery package within 10NM of this beacon, taxiway or windsock. In X-Plane these things must be attached to an airport, and within 10NM of the main runway.
The beacon, taxiway or windsock is omitted.
Can't parse <area #n or object id> in <filename> This is most likely caused by the presence of obscure pre-FS2002 opcodes in the source BGL file.
This area or library object will likely be incompletely converted, but any other areas or objects in the same file will be OK.
Can't parse compressed file <filename> The file is compressed and could not be converted.

Can't parse file <filename> This is most likely caused by a corrupt BGL file.
This file will not be converted.

Non-zero altitude (<n>m) for object <id> at (<lat>, <lon>) in file <filename> This object was placed in MSFS at <n> metres above ground level (AGL). In X-Plane, all scenery objects are placed at ground level.
You will likely need to edit the converted object to make it appear at the correct distance above the ground in X-Plane.
Non-zero pitch/bank (<pitch>/<bank>) for object <id> at (<lat>, <lon>) in file <filename> This object was placed in MSFS with non-zero pitch and/or bank. In X-Plane, all scenery objects are placed upright.
You will likely need to edit the converted object to make it appear correctly in X-Plane.
Object <name> is a built-in object The scenery package used a built-in "stock" MSFS scenery object.
The object is omitted.
Object <name> not found The scenery package used a library object that was not included in the scenery package.

Shortened ICAO airport code from <code1> to <code2> The scenery used an ICAO airport code longer than 4 digits. X-Plane only supports 4-digit ICAO codes.

Skipping animation in [<object id> in] file <filename> Animated scenery was found.
The scenery is converted as static scenery.
Skipping duplicate of taxiway at (<lat>, <lon>) in file <filename> This is probably caused by an error in the MSFS scenery package. The duplicate taxiway is ignored.
If you see lots of these errors, this may indicate that you have multiple alternative airport definitions installed. Read the installation notes that came with the scenery package, delete all but one of the alternative layouts, and run the application again.
Skipping misc data in file <filename> This is most likely caused by a request in the MSFS scenery package to flatten an area.
The request is ignored.
Skipping pre-FS2002 scenery in [<object id> in] file <filename> Old pre-FS2002 opcodes were found.
The old-style scenery is not converted, but other scenery in this file or library object will be OK.
Skipping pre-FS2004 runways and/or roads in [<object id> in] file <filename> Runways and/or roads were found in the old pre-FS2004 format. This application only handles FS2004 and FSX format runways (ie those authored with AFCAD2 or similar).
The runways and/or roads are ignored.
Skipping terrain data in file <filename> The scenery package includes terrain and/or landclass data.
This data is ignored.
Skipping traffic data in file <filename> The scenery package includes AI traffic data.

  • The file is one of the "stock" textures that are bundled with the MSFS program. Objects that use this texture will appear grey in X-Plane. You can fix this by providing a replacement for this texture.
  • This scenery package relies on another package or on a library package, which supplies the missing file. Read the installation notes that came with this scenery package to identify the missing package. Then copy the missing package into this package and run the application again.

rwy12 installation

Many scenery packages depend on the "rwy12" library. Unfortunately the installation notes for this library are a little confusing. You may find these notes easier to follow:

  1. Locate your Addon Scenery folder inside of your MSFS installation.
  2. Download and open rwy12_lib1_v1.2.zip.
  3. Copy the folder Static Objects Library into your Addon Scenery folder.
  4. Close rwy12_lib1_v1.2.
  5. Open the new Static Objects Library folder that you just created. You should see two sub-folders called scenery and texture.
  6. Download and open rwy12_lib2_v1.2.zip. Amongst other things it contains folders called scenery files and texture files.
    • Open the scenery files folder. Select all the files and copy them to the scenery folder inside the Static Objects Library folder that you just created.
    • Open the texture files folder. Select all the files and copy them to the texture folder inside the Static Objects Library folder that you just created. You may be asked whether to overwrite some files; say yes.
  7. Download and open rwy12_lib3_v1.2.zip and repeat step 6.
