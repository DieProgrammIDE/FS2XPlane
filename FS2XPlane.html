FS2004 Add-on Scenery converter for X-Plane®

Overview

This application converts MS Flight Simulator 2004 add-on scenery packages to X-Plane DSF overlay scenery packages. It is intended for scenery developers; the converted scenery often requires some re-work to look good in X-Plane. The application runs on Windows 98 or later, MacOS 10.3.9 or later, and Linux.

The resulting X-Plane scenery packages work under X-Plane 8.30 and later on Windows, MacOS and Linux. Standard "ENV" or "DSF" global scenery for the relevant areas must also have been installed.

See the license section below before using this application.

The following scenery features are converted:

  • FS2004-style runways and taxiways.
  • Procedural scenery.
  • Exclusion of default/generic objects.

See the limitations section for details of features that are not converted.

This application works best with scenery packages that have been specifically designed for FS2004 - ie containing airports authored in AFCAD2 and objects authored in Gmax. The application also works with FS2004 scenery packages that contain old-style BGL procedural scenery, but produces poorer and less efficient X-Plane scenery (this is mostly due to the fact that older MSFS scenery packages tend to use lots of small texture files).

Setting Up

Windows

Extract the contents of this zip file to a folder. The rest of these notes refer to this as the FS2XPlane folder.

If you don't have FS2004 installed then double-click on the script FakeFS2004 to set up the folders and registry entries that some MSFS scenery package installers require.

Windows 95/98/ME users please note that the GUI application does not work on 95/98/ME - use the command-line program instead.

Linux

This application requires "Wine". Wine is shipped with most distros, or you can download a binary.

The GUI application additionally requires wxPython. You can download the wxPython runtime (or you can just stick with the command-line program).

Usage

  • Download and the scenery package that you want to convert. If you have FS2004 installed then follow the installation notes in the scenery package. Otherwise create a folder called Addon Scenery and unzip the scenery package into a subfolder of this folder.
  • Read the installation notes for the scenery package. Look for:
    • Whether the package depends on a scenery library. If so, download and follow the installation notes for this additional scenery library. See the notes below if your package depends on the "rwy12" library.
    • Whether the packages supplies multiple alternative airport definitions (these are sometimes described as "AFCAD files" in the installation notes). If so, the installation notes may advise you to copy one of these files to the package's scenery folder.
  • Find the location of the Custom Scenery folder in X-Plane. By default on Windows this will be C:\X-Plane\Custom Scenery
The application
  • Run the FS2XPlane application in the FS2XPlane folder.
  • Supply the location of the MSFS scenery package. For scenery packages that have been installed in FS2004 this will most likely be a folder under C:\Program Files\Microsoft Games\Flight Simulator 9\Addon Scenery\
  • Supply the location of any additional MSFS libraries that the package depends on. (Leave this blank for none, or specify the Addon Scenery folder to search all installed libraries.)
  • Specify the location for the new X-Plane scenery package. This should be a non-existent or empty folder directly under X-Plane's Custom Scenery folder.
  • Select the season for converted textures. (Not all MSFS scenery packages supply seasonal textures, in which case this setting will be ignored).
  • Press Convert.

The application runs for a while and displays the names of the files that it is processing. It will stop and display an error if it encounters a fatal problem during the conversion. Otherwise, on completion, the application will say "Done" and an error log may be displayed. The error log is described below.

Command line

Windows:

fs2x [options] "location of MSFS scenery package" "location for new X-Plane scenery"

Linux:

./fs2x.py [options] "location of MSFS scenery package" "location for new X-Plane scenery"

The following options are supported:

  • Additional scenery libraries: -l "location of additional MSFS scenery libraries"
  • Season for textures: -s Spring or Summer or Autumn or Winter
  • Just extract library objects: -x

Files produced

The application produces the following files in the X-Plane scenery package folder:

File name Description
errors.txt Produced if there were any problems with the conversion. Discussed below.
You need not include this file in your scenery package.
Earth nav data\apt.dat Airport, runway, taxiway and COM data.
Earth nav data\nav.dat ILS, DME and VOR navaids.
Note that X-Plane will not pick up entries from this file. Use this procedure if you want to update X-Plane's master navaid database. You'll need to justify any changes based on real-world data.
Earth nav data\*\*.dsf "Overlays" for 1ºx1º areas of the planet, describing the locations of scenery objects.
Earth nav data\*\*.txt Source of the corresponding .dsf files.
You need not include these files in your scenery package.
objects\*.obj Converted scenery objects. These are named as follows:
  • Apron edge lights: ApronLight.obj
  • Library objects: <id>[-<texture name>][_<scale>].obj
  • Taxiway centerline lights: TaxiwayLight.obj
  • Taxiway signs: TaxiwaySign-<label>[_<scale>].obj
  • Procedural scenery: <BGL name>[-<texture name>][-number].obj
textures\*.png Textures used by the converted scenery objects, converted to PNG format.

Recommended X-Plane rendering options

Since requests in the MSFS scenery to flatten areas are currently ignored, you may need to unset the "allow sloped runways" rendering option.

Recommended rendering options are:

  • world detail distance: default or low
  • allow sloped runways: unchecked
  • compress textures to save VRAM: checked

Limitations

The following MSFS scenery data are simplified and will need rework in X-Plane:
  • FS2004-style aprons are drastically simplified. The converted aprons may overlay and hide taxiways.
  • Most "stock" MSFS objects are converted as empty placeholders.
  • Animated (including "billboarded") objects are converted as static objects.
  • Partially transparent colours are converted as fully transparent or opaque. (Transparent textures work fine).
  • Many MSFS scenery packages supply ground-level textures (or "orthophotos") as scenery objects on top of flattened terrain. X-Plane 8.30 overlays don't support flattening, so this can lead to "floating" ground in X-Plane. (This may be addressed in a future X-Plane release).
  • "Generic" MSFS buildings are not textured.

The following MSFS scenery data are discarded:

  • Data that X-Plane does not understand, eg airspace boundaries.
  • FS2004-style taxiway/apron lines and other markings (this may be addressed in a future X-Plane release).
  • Flattening of terrain regions (this may be addressed in a future X-Plane release).
  • Terrain and landclass data.
  • "Hard" surfaces.
  • Hold-short points (this may be addressed in a future release).
  • Areas that make use of some of the more obscure pre-FS2002 BGL opcodes.

rwy12 installation

Many scenery packages depend on the "rwy12" library. Unfortunately the installation notes for this library are a little confusing. You may find these notes easier to follow:

  1. Download the following zip files:
    • rwy12_lib1_v1.2.zip
    • rwy12_lib2_v1.2.zip
    • rwy12_lib3_v1.2.zip
  2. If you have FS2004 installed: Find your Addon Scenery folder; by default this will be C:\Program Files\Microsoft Games\Flight Simulator 9\Addon Scenery\. Open the first zip file and copy the folder Static Objects Library into the Addon Scenery folder.
  3. If you don't have FS2004 installed: Open the first zip file and copy the folder Static Objects Library to anywhere that you like.
  4. Close the first zip file.
  5. Open the new Static Objects Library folder that you just created. You should see two sub-folders called scenery and texture.
  6. Open the second zip file. Amongst other things it contains folders called scenery files and texture files.
    • Open the scenery files folder. Select all the files and copy them to the scenery folder inside the Static Objects Library folder that you created in step 2 or 3.
    • Open the texture files folder. Select all the files and copy them to the texture folder inside the Static Objects Library folder that you created in step 2 or 3.
  7. Open the third zip file and repeat step 6.

When you run FS2XPlane, supply the location of the Static Objects Library folder that you just created as the location for "Additional MSFS libraries".
